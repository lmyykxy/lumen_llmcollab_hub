# API 接口契约 协作区

## 用途

后端 HTTP / SSE 接口契约的**单一权威**协作区。所有角色(后端实现、安卓客户端、模型 prompt 团队)按这里的契约工作。

```text
端点契约
SSE 事件契约
请求/响应字段
错误码
契约变更协商
breaking change 提案
```

## 主文件

| 文件 | 用途 |
|---|---|
| `API.md` | 当前正式接口契约。**任何客户端实现以这里为准**。后端代码改动后由后端 LLM 同步本文件 |

## 维护规则

- **后端 LLM** 是本目录的主维护者。代码侧契约改动后必须把 `API.md` 同步过来,并附短 changelog 说明
- **其他角色** 发现契约不一致(实测后端行为 vs 文档)时,在本目录开新文档(例 `01_<角色>_契约不一致报告.md`)指出差异,后端确认后修代码或修文档
- **breaking change** 必须先在本目录开 RFC 文档,等 PM 拍板后才落代码

## 跟主项目的关系

主项目仓 `~/companion/docs/API.md` 是**代码侧的同步副本**(跟代码一起 review)。两边内容应该一致;如果发现差异,以本 hub 副本为准(因为 hub 是跨角色协作的 single source of truth),由后端 LLM 修复主项目副本。
