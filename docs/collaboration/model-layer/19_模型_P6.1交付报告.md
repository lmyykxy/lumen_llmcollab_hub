## 3. 改动详细

### 3.1 修正 "画我自己" 误判(PM §4.3)

**之前**(P6):`_XIAOQI_SELF_TRIGGERS` 含 "画我自己" — **错的**,把用户意图理解成"小七"。

**之后**(P6.1):
- 从 `_XIAOQI_SELF_TRIGGERS` **删除** "画我自己"
- 新加 `_USER_SELF_TRIGGERS = ("画我自己", "画我", "画一下我", "画一张我", "我的画像", "我的样子")`
- 新加 `SubjectType` 枚举 `"user_self"`(优先级在 xiaoqi 之前)
- 新加 `IntentType` `"user_self_request"`
- subject == "user_self" 时:
  - `reference_image_path = None`(不用 xiaoqi.png)
  - `identity_anchors = {}`(不写小七锚点)
  - 走 text2img 走通用 prompt(没用户视觉参考)

### 3.2 expression_intent + body_language_hint(PM §4.1)

**之前**(P6):`expression intensity` 在 `variable_elements` 里仅作 hint,**不进 prompt**。

**之后**(P6.1):
- `ImageIntent` 加字段 `expression_intent: str` + `body_language_hint: str`
- 7 类语义映射(对应 PM §4.1 给的方向):

| 用户关键词 | expression_intent | body_language_hint |
|---|---|---|
| 生气/不高兴/嘟嘴 | slightly annoyed, puffed cheeks, stubborn | arms slightly crossed or pouting |
| 害羞/不好意思/脸红 | shy, averting eyes slightly, faint blush | subtle leaning away or tucking chin |
| 委屈/难过/伤心 | quiet, slightly wronged, soft eyes (NOT dramatic) | shoulders slightly drawn in |
| 困/刚睡醒/睡眼 | sleepy, half-lidded eyes, soft morning | loose relaxed posture |
| 开心/高兴/得意/笑 | small proud smile, bright but restrained | natural upright posture |
| 嘴硬/别盯着看/羞 | embarrassed but stubborn, looking away | arms crossed or hands fidgeting |

- 用户没明确关键词时,从 `char_state.mood` / `hidden_mood` 兜底:
  - `hidden_mood="想被关心"` → "calm soft expression with subtle wistfulness"
  - `mood="低"` → "quiet calm expression, soft eyes (no dramatic sadness)"
  - `mood="高/兴"` → "small content smile, bright but soft"
  - 默认 → "calm, neutral-soft expression"
- `final_prompt` 拼接时插入两个新段:
  ```
  Expression: {expression_intent}.
  Body language: {body_language_hint}.
  ```

**关键约束**:
- 表情只改强度,**不改身份锚点**(尤其不改眼睛 — 由 negative_rules 兜底)
- 委屈不戏剧化(显式 "NOT dramatic crying")
- 不写媚俗 / 讨好 / 恋爱模拟器 风格

### 3.3 概率式 draw decision(PM §4.2)

**之前**(P6):LLM 一旦决定调 generate_image 就总是画,关系只影响构图距离。

**之后**(P6.1):新增 `DrawDecision` + `decide_draw()`,在 `_generate_image` executor 内:
1. 先 build ImageIntent
2. 再 `decision = decide_draw(intent, rel_state, user_message, rng=...)`
3. 如果 `decision.should_draw == False`:
   - 不调 image_gen provider(节省 LLM 成本 + 时间)
   - 返回 `{ok: false, declined: "not_in_the_mood", alternative_subjects: "..."}`
   - LLM 在 iter=1 看到 ok=false → 自然嘴硬岔开 / 建议替代画面

**算法**(`_classify_request_sensitivity` + `_draw_probability`):

```
sensitivity:
  普通场景 (window/room/object)    → low (基础概率 1.0,关系不影响)
  xiaoqi 普通画像 (画你/你的样子)   → medium (基础概率 0.6)
  xiaoqi + 高私密词 (自拍/正脸/床/穿) → high (基础概率 0.25)
  user_self / other                 → medium

prob = clamp(base + (trust - defense) / 100, 0.05, 1.0)
```

**典型场景预期值**:

| 关系状态 | xiaoqi medium 请求 prob | xiaoqi high 请求 prob |
|---|---:|---:|
| stranger 默认 (trust=0, def=60) | 0.05 (clamp 下限) | 0.05 |
| 中等-偏近 (trust=50, def=30) | 0.80 | 0.45 |
| 高亲密 (trust=80, def=10) | 1.00 | 0.95 |

**普通场景**:无论关系如何永远 1.0(`画窗外` / `画桌面` / `画一只猫` 不会被关系拦截)。

**单测 mock random**:`decide_draw(rng=lambda: 0.5)` 等可重复测试边界。

## 9. 已知未做的事

### 9.1 user_self 没视觉参考
当前 `subject="user_self"` 时不加载任何参考图,LLM 写的 raw_prompt 会得到一个泛二次元用户画像(可能跟用户实际样子完全无关)。
PM §4.3.2 提到 "若无法获得用户本人视觉参考,应走 user_self / other / insufficient_reference 分支" — 当前实现走 user_self 分支没参考,这是符合的。
真要画准用户,需要后续加 user upload reference 流程,**P6.1 不做**。

### 9.2 sensitivity 关键词扫不够智能
当前 keyword match,简单稳定。但偶发可能漏("正脸特写")或误判("画一张你穿衣服" — "穿"匹配高私密 + "衣服"也匹配)。
真要更智能,P6.5 可以上 LLM judge — 但成本翻倍。

### 9.3 概率公式偏简单
`base + (trust-defense)/100` 是线性叠加,可能在 high sensitivity 时太严(stranger 永远只有 0.05 概率画自拍)。
观察 1-2 周真实 user 行为后可以调整 base 或加入 sigmoid。**P6.1 不动**。

### 9.4 decline 后 LLM 自然圆场仰赖 prompt
我给 LLM 的 hint_for_assistant 是"现在不太想画这个,可以嘴硬岔开,或者建议画窗外/桌面",依赖 LLM 配合。
如果 LLM 仍然机械地说"图片生成失败",会被 P3 output_filters 拦下 — 双层保护。
