## 1. Background

P6.2 让"画你窗外"走 xiaoqi self → 注入 xiaoqi.png + 身份锚点。但实测 smoke 生成图为**小七正脸朝镜头**,窗户只是背景,跟"窗外"语义脱节:

```
P6.2 LLM 写的 raw_prompt(简单元素堆砌):
"view from a cluttered bedroom window, thick heavy gray clouds outside,
messy windowsill, soft warm interior lighting"
```

**问题诊断**:LLM 写 prompt 时只覆盖"场景元素",缺 4 个关键维度(动作 / 视角 / 构图 / 风格)。gpt-image-1 看到模糊指引就 fallback 默认 portrait。

## 2. P6.3 修复 — 工具描述加 6 维度引导

**改动**:`prompts/tools/generate_image.md` 的 `text2img` 段重写,加 6 个维度 + 反例 / 正例。

### 2.1 6 个必含维度

| 维度 | 含义 | 例 |
|---|---|---|
| **1 主体** | 画面焦点是谁/什么 | `"a cat-eared girl..."` 或 `"view from a window"` |
| **2 动作 / 视角** | 主体在做什么 + 镜头朝向 | `"leaning on the windowsill from behind, side profile, looking out"` |
| **3 场景** | 周围环境细节 | `"messy windowsill with sketchpads"` |
| **4 构图** | 远 / 中 / 近 + 主体占比 | `"wide composition, scene takes 60% of frame"` |
| **5 光照 / 气氛** | 色调 + 情绪 | `"soft melancholy afternoon light"` |
| **6 风格** | 视觉气质 | `"chibi-leaning illustration"` |

### 2.2 关键引导:含人就要写动作

```
画 "你 X 场景"(画你窗外 / 画你桌面)如果含小七,**必须写她在做什么**:
✅ "leaning on the windowsill from behind, side profile, looking out"
❌ 只写 "a girl in front of a window" → 默认正脸 portrait,跟"窗外"语义脱节
```

含/不含小七由 LLM 自己决定 — 看场景跟小七搭不搭。

### 2.3 给的反例 / 正例对比

**❌ 元素堆砌**(P6.2 LLM 实际写的):
```
"view from a cluttered bedroom window, thick heavy gray clouds outside,
messy windowsill, soft warm interior lighting"
```

**✅ 含小七 + 6 维度齐全**:
```
"a cat-eared girl with deep blue-purple long hair leaning on the
windowsill from behind, side profile looking out at thick gray clouds;
medium-wide composition with the window taking 60% of frame;
messy windowsill in foreground with sketchpads;
soft melancholy afternoon light;
chibi-leaning illustration style"
```

**✅ 纯景**(完全不含小七):
```
"view from a window, heavy gray clouds, a single bird flying past in
mid-distance, distant rooftops with rain shimmer;
wide-angle composition fully on the outside scene, no character in frame;
muted blue-gray melancholy lighting; soft illustration style"
```

## 5. 工作机制

```
用户:画一下你窗外
  ↓
Kimi(LLM)
  - 看 system prompt(状态、关系、人格)
  - 看 generate_image 工具描述(P6.3 新加 6 维度引导)
  - 决定调 generate_image,写**6 维度齐全的 raw_prompt**
  ↓
ImageIntentBuilder
  - 看 raw_prompt + user_message 识别 subject=xiaoqi
  - 注入 xiaoqi.png 参考图 + 身份锚点
  - 拼最终 prompt(LLM raw_prompt 已经丰富,后端只补身份锚点 + negative_rules)
  ↓
gpt-image-1
  - 看到 6 维度齐全的 prompt + xiaoqi.png 参考
  - 生成"小七侧脸看窗外"的画面(动作 / 构图 / 光照都对)
```

**关键**:LLM raw_prompt 越丰富,gpt-image-1 越知道镜头怎么放、主体在哪、要不要含人。后端 ImageIntentBuilder 只是锁身份 + 加 negative,不替 LLM 想构图。

## 6. 副作用 / 已知影响

### 6.1 LLM 偶发可能写不全 6 维度
真实场景 LLM 可能漏一两个维度(比如忘了风格)。当前工具描述用"必须"等强约束,基本覆盖。如果发现某维度高频缺失,加专门提醒。

### 6.2 prompt 字符数变长
原 100-400 字符建议 → 实测现在 raw_prompt ~600-700 字符,加身份锚点后 final_prompt 可能 ~1200 字符(还在 MAX_PROMPT_CHARS=1000 上限内,会触发截断)。
**未来可能要调高 MAX_PROMPT_CHARS** 或在 ImageIntentBuilder 里更紧凑拼接。本期不动。

### 6.3 Token 成本
LLM 写更长 prompt 多消耗少量 output tokens(~200 tokens 增量),可接受。
