## 6. P6.3.1 hotfix(commit 50d107c)— 用户验收期发现

### 6.1 现象

用户用 user 23 连续生成小七画像,小七越画越不像:猫耳帽消失/红围巾丢失/
身份漂移。

### 6.2 根因

日志显示 user 23 4 次 `generate_image` 都 `subject='room' has_reference=False`
——**完全没用 xiaoqi.png 当参考**:

| trace | user_message | subject | has_reference |
|---|---|---|---|
| 2467f128 | 能给我你的图片吗 | room | False |
| eae43b87 | 你还有照片吗,我觉得你长的好好看 | room | False |
| 1e6549ce | 我还想看一张嘛 | room | False |
| a124a06a | 最后一张,再发最后一张 | room | False |

两层叠加:
1. user_message 不含 PM §4 的 7 marker(用户用"你的图片"/"再发一张" 等承接语)
2. `_detect_subject` 把 user_message + raw_prompt 一起扫 ROOM_KEYWORDS,
   LLM 写的 "in cozy bedroom" 误把 subject 拐到 room

每次都没用参考图,小七靠 LLM 自己写的描述凭空生成,身份特征(猫耳/红围巾)
递归丢失。

### 6.3 修复方案(用户拍板)

不靠后端关键词反推 subject,**让 LLM 自己在 generate_image schema 里
直接填**:

```python
"subject": {
    "type": "string",
    "enum": ["xiaoqi", "user_self", "scene"],
    "description": (
        "**必填**。这次画的主体是谁:\n"
        "- xiaoqi:画你本人/含你入镜的场景。后端注入 xiaoqi.png 保证身份连续。\n"
        "- user_self:画用户自己。\n"
        "- scene:纯场景/物体/不含人物。不引用 xiaoqi.png。\n"
        "**判断不准就填 xiaoqi**(AI 伴侣场景默认画你最稳)。"
    )
}
"required": ["prompt", "subject"]
```

backend 拿 LLM 填的 `subject` 当主信号,keyword 路径降级成"LLM 漏填或无效填"
的安全网。

### 6.4 验证(重启 backend 后实测)

| smoke | input | LLM 填 subject | xiaoqi.png |
|---|---|---|---|
| HOT_REFER  | 你的图片有吗     | xiaoqi | ✓ 用 |
| HOT_FOLLOW | 再发一张吧       | xiaoqi | ✓ 用(承接语正确归小七) |
| HOT_PURE   | 画一只猫         | scene  | ✗ 不用 |
| S4 复跑    | 画一下你窗外现在的样子 | scene | ✗ LLM 创作选纯景 |

✅ user 23 漂移根因解决:LLM 知道是画自己 → 总能正确填 xiaoqi → 后端自动注入
xiaoqi.png

### 6.5 PM §4 一致性微调

PM §4 原文:"画你窗外 → 默认小七 + xiaoqi.png"是**软性建议**(用了"默认"
不是"强制")。P6.3.1 schema 让 LLM 有显式选择权:
- LLM 选 xiaoqi → 走 PM §4 默认路线
- LLM 选 scene → 用 PM §4 第 2 条"纯场景不用 xiaoqi.png"路线

S4 复跑实测 LLM 选 scene 走纯景,输出"喏。今天云压得很低。"自然到位。
**LLM 创作判断比硬规则更灵活**。

如果 PM 认为 PM §4 第 1 条要硬性("画你 X" 必须 xiaoqi),可以加后端 override
("user_message 含 PM §4 marker 时强制升级 LLM 的 scene 为 xiaoqi"),
但本期不做——先观察 LLM 主导后的整体质量。

## 8. PM 复审建议关注点

### 8.1 LLM 主导 subject 的边界

**问**:PM §4 "画你窗外 → 默认小七" 是软性建议还是硬性规则?

但本期我们倾向**先观察 LLM 自由判断的整体质量**,不加硬规则。

### 8.2 max=3 没真实触发

S7 实测 LLM 普遍 ≤ 2 skill,max=3 闸口没在 live smoke 触发。这是 LLM
自我节制的好结果,不是 bug。max=3 仍由 unit test 兜底(防极端情况)。

### 8.3 工具透明化 filter 的 false-negative

S4 trusted 漏出"我再确认一下画纯场景的规则"已加进 filter。但其它形态的
透明化(LLM 在 reasoning 里 leak skill 概念到 content)只能通过持续观察
+ 增量补 filter 词应对。**不会一次性穷尽**,接受这是观察期持续工作。

### 8.4 用户在 prompt 文件里加的硬约束

用户在验收期同时改了 `prompts/role/identity.md`("生成含你的图片时,必须
调用 xiaoqi.image.self skill")和 `prompts/skills/xiaoqi.image.self.md`
("不变元素后附加:The object must remain highly consistent with the
reference image.")。这两处改动**用户自己提交**,跟模型层 P6.3.1 commit
