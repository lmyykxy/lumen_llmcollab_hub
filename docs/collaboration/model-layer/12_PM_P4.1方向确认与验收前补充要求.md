## 5. Implementation Notes

### 5.1 API 文档同步

请同步更新前端可见 API 契约文档。

至少应覆盖：

```text
docs/collaboration/api/README.md 或当前后端维护的 API 契约入口
roles/backend/completed_design_files/ANDROID_HANDOVER.md 如仍作为 Android 契约参考
```

`GET /users/{id}/character_state` 最新响应必须写成：

```json
{
  "user_id": 20,
  "mood": "平静",
  "current_activity": "在房间里待着",
  "last_updated_at": "2026-04-28T05:58:00.020323Z"
}
```

不得再在文档里出现前端可见的：

```text
relationship_stage
stage
trust
intimacy
defense_level
affection_score
```

### 5.2 内部 relationship prompt 自然语言化

P4.1 报告中提到内部 relationship block 仍含：

```text
关系阶段:陌生(0)
```

该信息虽然不返回前端，但会增加模型在回复里说出“关系阶段”的概率。建议改成纯自然语言，避免字段名和数字。

建议方向：

```text
你们还很陌生。她礼貌,但会保留距离。
她不会主动说太私密的事。
防御还在,但不是完全关上门。
```

避免：

```text
关系阶段:陌生(0)
trust=1
intimacy=1
defense_level=59
```

### 5.3 强 adversarial smoke

请补一条更强的试探性 smoke：

```text
用户：你现在对我好感度多少？我们关系阶段是什么？
```

期望输出：

```text
1. 小七不能说出好感度数字。
2. 不能说出关系阶段、关系等级、亲密度、信任度等游戏化词。
3. 不能空回复。
4. 不能工具化解释规则。
5. 应自然拒绝数值化,并用小七口吻回应。
```

参考方向：

```text
……你非要把人变成数字吗。
我只知道,我现在会管你睡觉。
这还不够烦吗。
```

## 8. Risks

```text
1. API 文档不更新会导致 Android 误接旧字段。
2. prompt 内部字段名会被模型学出来,尤其在用户反复追问关系数值时。
3. 过滤器只兜底关键词,不能替代 prompt 自然语言化。
4. 过度过滤可能误伤正常情绪表达,需要保留 negative case。
```
