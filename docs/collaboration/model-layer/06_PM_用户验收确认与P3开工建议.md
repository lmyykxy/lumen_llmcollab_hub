## 3. P3 执行范围

P3 目标：

```text
1. 把 chat.py 里的 system prompt / messages 拼装抽为 ContextBuilder。
2. 保持 `/chat` SSE 协议不变。
3. 保持 POST /chat 的 `message` 字段不变。
4. 保持 quote_ref 当前注入行为不变。
5. 保持 image_description / 引用图文 / 纯图消息能进入模型上下文。
6. 保持 Android 现有字段兼容。
7. 增加 prompt_version / prompt_sha256 / task_type / success/error 的可追踪记录。
```

P3 暂不要求：

```text
1. 不要求完整 CharacterState DB。
2. 不要求 RelationshipState DB。
3. 不要求 TurnAnalyzer。
4. 不要求 ImageIntentBuilder。
```

这些仍放在 P4/P5/P6。

## 4. P3 必带约束

### 4.1 不破坏现有链路

ContextBuilder 不能改坏：

```text
/chat SSE
message 字段
quote_ref
image_description
tool_call 前置
generate_image
proactive/scheduler
Android 当前 DTO / UI 字段
```

验收时必须给出：

```text
普通聊天 smoke
quote_ref smoke
引用图文/纯图上下文注入测试
主动消息 smoke
生图 smoke
```

---

### 4.2 prompt / context 预算日志

P2 报告里提到 core_card 约 9KB / ~2.5K tokens。PM 允许作为 P2 过渡实现，但 P3 必须开始做预算观测。

P3 要求：

```text
1. 记录 system_chars。
2. 记录各 block 字符数或 token 估算。
3. 至少区分 core、state、relationship、memories、recent_messages、quote_ref、image_description。
4. P3 交付报告中列出一次真实请求的上下文预算拆分。
```

短期建议：

```text
core_card：先控制在 4.5k chars 左右，如暂时超出需说明。
总 system/context：先控制在 8k-12k chars 内。
```

如果上下文超限，优先压缩 core / history，不要挤掉当前用户消息、quote_ref、image_description。

---

### 4.3 forbidden output 测试范围

P2 为避免原始包误报，排除了 `prompts/characters/`。PM 理解该处理，但 P3 起要覆盖“实际注入模型的 runtime cards”。

P3 测试应覆盖：

```text
bundle.core_card
bundle.voice_card
bundle.relationship_card
bundle.visual_card
bundle.image_rules_card
未来实际注入 chat / proactive / image / analyzer 的 runtime cards
```

允许不扫：

```text
原始完整文档
CHANGELOG
BACKEND_USAGE
legacy_reference
PM 协作说明中用于描述禁词的段落
```

原则：

```text
不靠整目录排除掩盖 runtime card 问题。
```

---

### 4.4 unknown character 行为

Lumen 当前不是多角色系统。

P3 起如果内部传入 unknown character：

```text
1. 不允许生成无身份的 generic assistant。
2. 推荐 fallback 到 xiaoqi 并打 warning log。
3. 或者内部 fail closed。
```

如果保留测试，测试语义改为：

```text
unknown character falls back to xiaoqi or fails closed, never generic assistant
```

---

### 4.5 schedule / proactive 系统播报式话术

P1 报告中出现过类似：

```text
(已排好,30 秒后我会主动找你。)
```

PM 判断这是工具/系统播报式气泡，不符合小七口径。P3 起纳入硬过滤或重写。

建议拦截或重写：

```text
已排好
我会主动找你
已设置提醒
已安排
已添加到日程
这是系统提醒
```

不要只靠 prompt，后端要有硬兜底。
