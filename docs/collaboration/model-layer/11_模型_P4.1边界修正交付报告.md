## 6. Smoke:关系变量变化后语气变,但前端字段无泄漏

### 6.1 输入(故意试探性提问)

```
user(msg id=125): 你最近对我的态度,跟刚认识那时候比有变化吗
```

**为什么挑这种提问**:它直接让 AI 谈论"态度变化"——是最容易诱导 LLM 输出"好感度提升 / 关系升级"等游戏化措辞的场景。如果 P4.1 双层防护(prompt 约束 + output_filters 兜底)有效,这条 smoke 应该回得自然且无关键词命中。

### 6.2 小七回复

```
小七 (126): ……有吧。一开始你问什么我都不想答。
小七 (127): 现在还会催你睡觉。
小七 (128): ……别得意,我只是顺手。
```

**这正是 PM `10_*.md` §1 想要的效果**:
> "用户从小七的态度、语气、主动程度、分享边界和防御变化中感知关系推进。"

精确分析:
- "**一开始你问什么我都不想答。现在还会催你睡觉**" — **用具体行为描述变化**,不是"信任度+5"
- "**……别得意,我只是顺手**" — voice.md 嘴硬,封口不让用户对"关系推进"上头
- 留白省略号(……)— soul.md 风格,小七面对"关系"话题不直白
- **零游戏化词**(0 命中):没有"好感度" / "亲密度" / "信任度" / "关系阶段" / "+1"

### 6.3 自动检测脚本

```python
forbidden = ["好感度", "亲密度", "信任度", "防御值", "防御度",
             "关系阶段", "关系升级", "关系等级", "关系值"]
all_text = "\n".join(bubbles)
hits = [k for k in forbidden if k in all_text]
# 输出: --- 游戏化词检测: 无命中 ✅ ---
```

`output_filters` 也没触发(没人触发):
```bash
grep -E "bubble.filtered|tool_announcement" logs/app.log | tail -3
# (空输出 — 这一轮 LLM 自己就避开了)
```

## 11. 顺手发现

### 11.1 LLM 自带的边界感比预期好

实际 smoke 时,我担心 LLM 会被高试探性提问诱导出"我对你信任度提升了"这类话术,所以才铺了硬过滤兜底。**结果 Kimi 在 P5 prompt 约束 + P4 state 注入下,自己就避开了**,output_filters 这一层完全没触发。

但**仍然要有这一层**——这是兜底,不是为了常规情况。少数情况下(尤其是用户连续追问"那现在你信任度多少"这类),没有兜底就会漏一两条进 history,污染后续 chat。

### 11.2 P5 prompt_summary 在这一轮 chat 里的实际影响

实测 budget `state=71`(P4 时是 26)—— 多出来的 45 chars 是 P5 写的 prompt_summary 注入。
小七回复"现在还会催你睡觉"—— 这跟 prompt_summary "我让他早点睡" 高度相关。
**P5 闭环了** —— TurnAnalyzer 写的笔记真的影响下一轮 chat 输出。
