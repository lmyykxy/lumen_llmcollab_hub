## 1. Background

模型/后端已交付 P6 ImageIntentBuilder，并证明：

```text
1. 小七本人画像能引用 `/root/companion/backend/res/xiaoqi.png`。
2. 普通聊天不会误触发生图。
3. generate_image executor 已接入 ImageIntentBuilder。
4. 关系变量目前能影响画面距离感和构图。
```

但用户补充确认了两个关键点：

```text
1. 生成小七图片时，提示词需要描述小七表情，并根据用户语义出现不同表情。
2. 用户和小七关系疏远时，不应基本每次都画图；应有较大概率不触发画图，由模型结合亲密度/关系状态判断。
```

PM 复核 P6 报告后判断：P6 方向可保留，但不建议直接验收。需要先做 P6.1 小修正。

---

## 2. Scope

P6.1 只处理三类问题：

```text
1. 小七画像的语义表情必须进入最终生图 prompt。
2. 用户关系疏远/小七防御较高时，增加概率式 draw decision，不是每次都画。
3. 修正 `画我自己` 被识别为小七本人的误触发。
```

---

## 3. Non-goals

P6.1 不做：

```text
不新增前端 API 字段。
不向前端返回 trust / intimacy / defense_level / affection_score / relationship_stage。
不在 UI 展示好感度、亲密度、关系等级或抽卡式概率。
不把小七写成生图客服。
不硬禁止疏远关系下的所有生图。
不改变 `/chat` 的 message 字段、SSE 事件和 Android 现有兼容字段。
不把 `/root/companion/backend/res/xiaoqi.png` 路径返回给前端。
```

## 4. Implementation Notes

### 4.1 表情必须从 hint 升级为 prompt 约束

P6 报告目前写明：

```text
variable_elements 里有 expression intensity。
但可变元素仅作 hint，不进 prompt。
```

这不足以满足用户要求。P6.1 需要新增或等价实现：

```text
expression_intent
body_language_hint
```

要求：

```text
1. 用户明确要求表情时，final_prompt 必须包含对应表情描述。
2. 用户没有明确表情时，允许结合 user_message、CharacterState.mood、hidden_mood、relationship state 自然推断。
3. 表情只能改变强度和状态，不能改变固定身份锚点，尤其不能改变眼睛身份。
4. 表情描述应服务于小七主体性，不要写成媚俗、讨好或恋爱模拟器风格。
```

建议语义映射：

```text
生气 / 不高兴       → slightly annoyed, puffed cheeks, stubborn expression
害羞 / 不好意思     → shy, averting eyes slightly, faint blush, trying to look calm
委屈 / 难过         → quiet, slightly wronged expression, soft eyes, not crying dramatically
困 / 刚睡醒         → sleepy, half-lidded eyes, loose posture, soft morning expression
开心 / 得意         → small proud smile, bright but restrained expression
平静 / 日常         → calm, neutral-soft expression
嘴硬 / 别盯着看     → embarrassed but stubborn expression, looking away slightly
```

注意：这些是语义方向，不要求逐字使用英文。

### 4.2 增加关系感知的 draw decision

P6 当前做法是：

```text
LLM 已决定调用 generate_image 后，ImageIntentBuilder 再加工 raw_prompt。
RelationshipState.defense_level 只影响构图距离。
```

这不满足“疏远时有概率不画”的产品目标。P6.1 需要在工具调用前或执行前增加 draw decision：

```text
draw_decision / image_consent_gate / draw_willingness
```

该判断可以由模型完成，也可以由轻量策略 + 模型语义判断共同完成，但必须满足：

```text
1. 判断依据包括用户请求敏感度、最近上下文、小七当前生活状态、内部关系状态。
2. 关系疏远或防御高时，不是硬拒绝，而是更大概率不画小七本人、自拍、正脸、私密房间等高私密度画面。
3. 不画时，小七要自然回应，可以嘴硬、岔开或给替代画面建议，例如画窗外、桌面、草稿纸、房间一角。
4. 关系较近时，小七更可能愿意画自己或更私密的生活片段，但仍保持小七主体性，不变成奖励机制。
5. 任何概率、分数、阶段都只能在服务端内部或日志中出现，不得进入前端 response 或 caption。
```

建议内部策略：

```text
低亲密 / 高防御 + 小七本人/自拍/正脸/卧室私密请求：
  大概率不画，或改成更远、更克制的替代画面。

中等关系 + 小七本人请求：
  可画，但构图更远，表情更克制，少正面亲密姿态。

高亲密 / 低防御 + 小七本人请求：
  更可能画，允许更自然的房间、穿搭、表情和生活片段。

非小七本人主题，例如窗外、桌面、物体：
  关系影响应较弱，不应因为疏远而频繁拒绝普通场景图。
```

测试中如需稳定复现概率，可给 gate 注入可控随机种子或 mock random。

### 4.3 修正 `画我自己` subject 误判

P6 报告把 `画我自己` 放在 xiaoqi self triggers 里，这是语义错误。

PM 拍板：

```text
`画我自己` 默认指用户自己，不是小七。
```

P6.1 要求：

```text
1. 从 xiaoqi self triggers 移除 `画我自己`。
2. 若无法获得用户本人视觉参考，应走 user_self / other / insufficient_reference 分支，而不是使用 xiaoqi.png。
3. `画你`、`画小七`、`你的样子`、`你在房间里` 仍应识别为小七本人画像，并使用 `/root/companion/backend/res/xiaoqi.png`。
```

## 6. Acceptance Criteria

P6.1 交付报告必须包含：

```text
1. `画我自己` 不再使用 xiaoqi.png 的测试。
2. `画你` / `画小七` / `你的样子` / `你在房间里` 仍使用 xiaoqi.png 的测试。
3. “画一张你生气的样子” final_prompt 含明确生气/嘴硬表情描述。
4. “画你刚睡醒的样子” final_prompt 含困倦/刚睡醒表情和姿态描述。
5. “画一张你委屈的样子” final_prompt 含克制委屈表情，不走情感勒索。
6. 低亲密/高防御 + “画张你自拍” 的 smoke：多次或固定种子下大概率不实际调用生图，并以小七口吻自然拒绝/转移/给替代画面。
7. 高亲密/低防御 + 同类请求的 smoke：更可能触发生图，并仍使用 xiaoqi.png。
8. 中等关系 + 小七本人请求：可画但构图距离更克制。
9. 普通场景请求“画你窗外”不应被关系疏远过度拦截，也不应使用 xiaoqi.png。
10. 可见回复不得出现“好感度、亲密度、关系阶段、概率、抽中、权限、已解锁、图片已生成”等工具化/游戏化词。
11. API / SSE 兼容性说明：不破坏 `/chat`、message、quote_ref、image_description、Android 现有字段。
12. 单测结果和 smoke 全文。
```

---

## 7. Risks

```text
1. 如果 gate 过强，小七会显得不合作，影响正常使用。
2. 如果 gate 过弱，疏远关系下仍会频繁画自拍，关系变化感不足。
3. 如果表情映射过度，会让小七变成夸张表情包，破坏安静、嘴硬、轻微委屈的气质。
4. 如果概率或关系判断进入可见文案，会破坏“内部好感度不给前端”的底线。
5. 如果 `画我自己` 继续误判，会错误使用小七参考图生成用户画像。
```
