## 1. Background

P6 实施时,我加了 `_POV_SCENE_OVERRIDES`,把"画你窗外/画你的房间"等强制 override 到 `subject=window/room`,**不**用 xiaoqi.png。
理由是当时认为"画你窗外"是 POV 场景(从小七视角看的窗外),不是小七肖像。

P6.1 验收期间用户审视实际生成图,发现:

```
Smoke C 输入:画一下你窗外现在的样子
ImageIntent: subject=window has_reference=False
生成图:一个深色短发女孩坐在窗边(非小七!)
```

用户判断:**这是最差的结果**——不是小七,也不是纯窗外景,而是"像小七但又不是小七"的泛二次元女孩。

---

## 2. 用户拍板:走 B(回归 xiaoqi self)

我向用户提了三个选项:

| 方向 | 做法 |
|---|---|
| A | window/room 加 "no people" negative rule(纯环境无人物) |
| **B** | 删 POV override,"画你 X" 都走 xiaoqi self,用 xiaoqi.png 锁身份 |
| C | POV override 只匹配纯 POV 句式("从你窗外/你看见的窗外") |

**用户选 B**。

理由(我的分析):
- 中文"画你 X" 在自然语义里更接近"画**小七**在 X 的画面",不是"画 X 这个景物"
- PM 列的 xiaoqi self triggers 含 "**你在房间里**" / "**你今天穿什么**"——同样含场景但仍是小七为主体
- "画你窗外" / "画你的房间" 跟它们一类
- B 方向后:生成图既有小七又有场景(最自然语义),且 xiaoqi.png 锁身份不会漂

---

## 3. 跟 PM `16_*.md` §6.9 拍板的冲突

PM `16_*.md` §6.9 明确写:

> 9. 普通场景请求"画你窗外"不应被关系疏远过度拦截,**也不应使用 xiaoqi.png**。

P6.2 改动后,"画你窗外" 走 xiaoqi self → **会**使用 xiaoqi.png → **跟 PM §6.9 冲突**。

**模型 LLM 的判断**:
- 用户审视实际生成图后给出反馈,优先级高于 PM 文档拍板
- PM §6.9 当时可能没充分考虑"画你窗外"的中文歧义
- 走 B 的代价:跟 PM 拍板正面冲突 → 需要 PM 重审 §6.9
- 不走 B 的代价:用户继续看到"非小七的女孩"在窗边,产品体验最差

我选了 B,但**显式标记冲突**留给 PM 重审。

## 4. 改动详细

### 4.1 image_intent_builder.py

```python
# 之前(P6/P6.1):
_POV_SCENE_OVERRIDES = (
    ("你窗外",  "window"),
    ("你的窗",  "window"),
    ("你窗边",  "window"),
    ("你房间外", "window"),
    ("你的房间", "room"),
    ("你房间里的桌", "room"),
    ("你桌上",  "room"),
)

# 之后(P6.2):
_POV_SCENE_OVERRIDES = ()   # 清空
```

`_detect_subject` 优先级链不变(POV-override 还在,但因为 tuple 空所以走不到):
```
POV override (empty) → user_self triggers → xiaoqi triggers → 普通场景 → other
```

效果:`画一下你窗外的样子` 含 `画一下你` → 命中 `_XIAOQI_SELF_TRIGGERS` → subject=xiaoqi。

### 4.2 不动的事

- `_XIAOQI_SELF_TRIGGERS` 不变
- `_USER_SELF_TRIGGERS` 不变("画我自己" 仍走 user_self)

- expression_intent / DrawDecision 不动
- API / DB / chat / scheduler 0 改动

**这正是 P6.1 + P6.2 协作的预期**:
- P6.2 修对了 subject(画你窗外 → xiaoqi)
- P6.1 gate 看 stranger 关系(trust=1)→ medium sensitivity prob=0.05 → 95% 拒画
- 小七首句 "**笔还热着。你倒是上瘾了。**"(嘴硬:才刚画过自画像现在又来)
- iter=1 圆场遇 Kimi ConnectTimeout(网络问题,跟 P6.2 无关)
