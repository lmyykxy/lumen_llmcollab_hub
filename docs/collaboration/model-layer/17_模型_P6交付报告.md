## 3. ImageIntentBuilder 模块说明

### 3.1 职责

```text
1. 判断当前 turn 是否真的需要生图 → LLM 决策(我们的工具调用模型)
   ImageIntentBuilder 只在 LLM 已经决定调 generate_image 后,加工 LLM 给的 raw_prompt
2. 将用户消息 + 最近上下文 + CharacterState + RelationshipState + visual rules 组成 ImageIntent
3. 输出稳定 final_prompt + 选定 reference_image_path,供 generate_image executor 使用
4. 给日志提供可追踪 intent 摘要,但不把内部 prompt 或关系变量返前端
```

### 3.2 ImageIntent 字段(对照 PM §5.2)

```python
@dataclass(frozen=True)
class ImageIntent:
    intent_type:          IntentType        # user_request / xiaoqi_self_scene / diary_visual / album_visual
    subject:              SubjectType       # xiaoqi / window / room / object / other
    scene:                str               # 场景文本
    mood_phrase:          str               # state→画面措辞,自然语言
    style_notes:          str               # 距离/构图措辞
    identity_anchors:     dict[str, str]    # subject==xiaoqi 时填:hair/hat/eyes/facial_atmosphere/body_type
    reference_image_path: Path | None       # 仅小七本人画像非 None
    variable_elements:    tuple[str, ...]   # glasses/clothing/pose/expression intensity/lighting/scene
    negative_rules:       tuple[str, ...]   # 不改身份/不改眼睛/不画 generic 二次元
    caption_tone:         str               # 给后续 caption 阶段的提示
    final_prompt:         str               # 真正给 gpt-image-1 的拼好的英文 prompt
```

### 3.3 subject 识别策略(PM §5.4 触发关键词)

```text
优先级:POV 场景覆盖 > xiaoqi self triggers > 普通场景 > other

POV override(关键创新):
  "你窗外" / "你的窗" / "你房间外" / "你的房间" / "你桌上"  → window/room
  这些是从小七视角看的场景,不是小七肖像。
  避免把 chibi xiaoqi 头像贴到风景画里(实测中 LLM 偶尔会这样误调)。

xiaoqi self triggers(对齐 PM §5.4 + 常见语法变体):
  画你 / 画一张你 / 画一下你 / 画下你 / 画一幅你 / 画幅你 / 画张你
  画小七 / 画一张小七
  你的照片 / 你的样子 / 你现在的样子 / 你今天的样子
  你的头像 / 你在房间 / 你今天穿 / 小七自拍
  看看你 / 给我看看你 / 你长什么样 / 画我自己
  英文 LLM:self portrait / yourself / of xiaoqi / xiaoqi herself
```

### 3.4 state → 画面措辞(关键:不暴露字段名/数值)

PM §5.5 给的方向:
- 心情低 → 光线/构图/色调变安静,**但不能情感勒索**
- 关系更近 → 房间/手边物品/私人生活片段
- 防御更高 → 画面更远 / 克制

实现的 mood→视觉转译:
```python
"低/丧/累" → "soft, muted color palette; gentle low-key lighting" + "calm and quiet, not desperate or pleading"
"高/兴/好" → "warm, slightly saturated palette; bright natural lighting"
"懒/倦"   → "cozy soft palette; gentle afternoon-light feel"
默认       → "balanced natural palette and lighting"
```

defense_level→构图距离:
```python
>= 70 → "wider, more distant composition; less direct framing; no overly intimate angle"
>= 40 → "balanced composition; natural angle, neither too close nor too distant"
>= 20 → "closer, more comfortable composition; natural at-home framing"
< 20  → "intimate close composition; very relaxed, at-home angle (still not romantic-flirty)"
```

**关键约束**:`final_prompt` 不出现 "trust=" / "intimacy:" / "stranger" / "(0)" / "好感度" / "亲密度" / "关系阶段" 等字符 — 单测 17 个参数化 case 验证。

### 3.5 视觉身份硬约束(PM §5.3)

```python
IDENTITY_ANCHORS_EN = {
    "hair":              "deep blue-purple long hair with bangs",
    "hat":               "blue cat-ear cap with red trim",
    "eyes":              "purple-blue cat-like big eyes",
    "facial_atmosphere": "soft, cute, slightly innocent",
    "body_type":         "petite",
}
```

固定锚点写进 `final_prompt` 时附带:
> "Match these features carefully — do not generate a different anime girl."

可变元素仅作 hint(不进 prompt):glasses / clothing / pose / expression intensity / lighting / scene。

negative_rules 写进 prompt 末尾:
- do not change core identity
- do not alter eye shape or color
- do not generate generic anime girl
- no text, no watermark, no UI elements

## 4. 与现有 generate_image 工具接入

### 4.1 改动位置

`app/agent/tools/image_gen.py::_generate_image` 在 prompt 验证后 + reference 加载前插入 ImageIntentBuilder.build():

```python
# P6 加工
char_state = await get_or_create_character_state(ctx.db, ctx.user_id)
rel_state = await get_or_create_relationship_state(ctx.db, ctx.user_id)
try:
    intent = ImageIntentBuilder().build(
        raw_prompt=raw_prompt,
        user_message=ctx.user_message,
        char_state=char_state,
        rel_state=rel_state,
    )
except ImageIntentError as e:
    # PM §5.4.5: fail-closed — 小七本人画像但参考图不可读 → 拒,不生成"新小七"
    return json.dumps({"ok": False, "error": f"intent fail-closed: {e}"})

prompt = intent.final_prompt   # 替换 LLM 给的 raw_prompt
# reference 选择优先级:小七 ref > LLM 给的 reference_image_url > 无
```

### 4.2 关键路径分支

```text
intent.subject == "xiaoqi":
  → reference_bytes = read xiaoqi.png(645KB)
  → ref_source = "xiaoqi"
  → 走 OpenAI /images/edits (img2img)
  → LLM 写的 reference_image_url 被覆盖(避免误把用户上传图当小七参考)

intent.subject != "xiaoqi" 且 LLM 提供 reference_image_url:
  → 走原 P3 时代的 img2img 流程(用户上传图改色/改风格)
  → ref_source = "user_upload"

intent.subject != "xiaoqi" 且无 ref:
  → text2img,走 OpenAI /images/generations
  → ref_source = "none"
```

### 4.3 ToolContext.user_message 字段(P6 新加)

`app/agent/chat.py::ToolContext` 加了 `user_message: str = ""` 字段,
`stream_reply` 创建 ctx 时把 user_message 带上。这样 ImageIntentBuilder 在
executor 内能读到原始用户输入做 subject 识别,**不用查 DB**。

proactive_turn 没用户消息,默认空串 — `_detect_subject` 会落到 `other`,不触发 xiaoqi 参考图(主动消息不会突然画小七)。

## 9. 已知限制(对照 PM §5.4.3 的诚实声明)

### 9.1 OpenAI gpt-image-1 reference 锁身份效果未保证

`/images/edits` 端点把 reference image 当作"基础图",根据 prompt 改造。
对**身份一致性**(脸型/眼睛/发色精确还原)效果是 best-effort,**不是硬保证**。

PM §5.4.3 明确要求:"如果当前 provider 暂不支持参考图,交付报告必须明确说明限制,并先用视觉锚点 prompt 兜底,不得声称已经做到了参考图一致性。"

P6 同时做了**两层保护**:
- (a) reference image 直接给 provider(可能锁身份,可能只锁色调/发型)
- (b) prompt 里写明 IDENTITY_ANCHORS_EN(明确告诉模型固定特征)
- (c) negative_rules 写明 "do not generate a different anime girl"

如果 provider 升级到能锁身份(比如 gpt-image-2 + character reference),P6 不需要改代码,行为自动变好。

### 9.2 subject 识别是 keyword 匹配,不是 LLM 判别

简单 + 稳定 + 0 额外 LLM 调用。但偶发:
- 用户说"画一只像你那样的猫" — 含"像你",可能误触发?(实测:含"画"+"猫"会先走 _WINDOW/_ROOM 关键词扫描,猫不在场景列表,落到 other)
- 用户用拼音 / 表情 / 隐喻表达 → 当前漏掉

预防措施:
- POV 场景覆盖兜底("你窗外" 等明显场景词)
- 真出问题用户会重新表达,LLM 二次 tool_call

如果发现误触发率高,P6.1 可以升级为 LLM 判别(成本 ~1 次额外 LLM 调用)。

### 9.3 mood / hidden_mood 转译是 keyword 匹配

跟 9.2 同。当前足够。
