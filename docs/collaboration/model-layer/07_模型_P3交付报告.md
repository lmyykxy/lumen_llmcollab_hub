## 3. ContextBuilder 改动说明

### 3.1 模块拆分

| 之前(P2) | 现在(P3) |
|---|---|
| `chat.py::_build_system_prompt` | `app/agent/context_builder.py::ContextBuilder.build_chat` |
| `scheduler.py::_build_system_prompt` | `app/agent/context_builder.py::ContextBuilder.build_proactive` |
| 各自 `get_user_history` + `merge_assistant_bubbles` + render | 都收口到 ContextBuilder,返回 `ContextBundle` |
| 各自硬编"xiaoqi/v0.3.1" | ContextBuilder 构造时配置 |

### 3.2 ContextBundle 形状

```python
@dataclass(frozen=True)
class BlockBudget:
    core:                int
    memories:            int
    history_total:       int
    quote_ref:           int   # 估算:format_quoted_context_prefix 输出长度
    image_description:   int   # 估算:image_desc 注入长度(user suffix + assistant tool_body)
    proactive_trigger:   int = 0
    state:               int = 0  # P4 占位
    relationship:        int = 0  # P4 占位
    system_total:        int = 0

@dataclass(frozen=True)
class ContextBundle:
    messages:        list[dict]   # 直接传给 chat_provider.chat_stream
    blocks:          BlockBudget
    prompt_version:  str           # "xiaoqi/v0.3.1"
    prompt_sha256:   str
    total_chars:     int
```

### 3.3 chat.py / scheduler.py 接入

**chat.py::stream_reply**(第 ~325 行附近):
```python
ctx_bundle = await ContextBuilder().build_chat(
    db, user_id, retrieved, history_limit=HISTORY_LIMIT,
)
messages = list(ctx_bundle.messages)
tlog.info("event=chat.prompt_loaded version=%s sha256=%s system_chars=%d", ...)
tlog.info("event=chat.context_budget %s history_msgs=%d total=%d",
          format_budget_log(ctx_bundle.blocks), ...)
```

**scheduler.py::proactive_turn** 同模式,改用 `build_proactive(kind, context, ...)` —— PM §4.2 的 `proactive_trigger` 字段在这里被填充。

### 3.4 为什么 "预算" 用字符不用 token

- token 估算依赖具体 tokenizer,Kimi/OpenAI/Claude 各家都不同
- 中文字符数跟 token 数比例稳定(中文 1 字 ≈ 1.5-2 tokens),做趋势观测够用
- 真要 token 数,P5 TurnAnalyzer 阶段再上 tiktoken / Moonshot 的 token counter

## 10. 顺手发现的事

### 10.1 一次 chat 跑出 finish=length / bubbles=0(无 hallucination,但 API 浪费)

trace=404d6720(智 smoke 5 第一次): `finish=length elapsed_ms=39623 content_chars=0 reasoning_chars=4236 tool_calls=0`。
LLM 把所有 token 烧在 reasoning(Kimi thinking 模式),没产出 content / tool_calls。`max_tokens=2000` 被 reasoning 吃光 → finish=length。

不影响功能(用户重发后 trace=c0ee7307 正常生图),但每次发生都浪费一次 LLM 调用。建议 P5/P6 加 reasoning_chars 监控,触发率高时考虑提 max_tokens 或者 prompt 引导提前出 content。本期不动。

### 10.2 Moonshot 账户 suspend → 充值后恢复

P3 中段实测时 Moonshot chat 账户因余额不足触发 429 suspend。PM 充值后所有 5 smoke 全过。**Lumen MVP 强依赖 LLM 余额**,建议 PM 加 monitoring。

### 10.3 安卓友好性

`event=chat.context_budget` 当前是 stdout INFO log。后端 RPT-002 已经提过观测面板规划。这条日志结构化好,P5 上 grafana / observability stack 时可以直接做 budget 趋势图。
