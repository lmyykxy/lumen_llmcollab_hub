## 5. P6 Implementation Notes

### 5.1 新模块建议

建议新增：

```text
app/agent/image_intent_builder.py
```

职责：

```text
1. 判断当前 turn 是否真的需要生图。
2. 将用户请求、最近上下文、CharacterState、RelationshipState、visual rules 组织为 ImageIntent。
3. 输出稳定的工具参数,供 generate_image 使用。
4. 给日志提供可追踪的 intent 摘要,但不把内部 prompt 或关系变量返回前端。
```

### 5.2 ImageIntent 建议形状

```text
intent_type        user_request / xiaoqi_self_scene / diary_visual / album_visual
subject            小七本人 / 窗外 / 房间 / 用户给的物体 / 其他
scene              房间、窗边、夜晚、雨、桌面等
mood               来自公开 mood + hidden_mood 的自然语言转译
style_notes        小七视觉风格,不能泛化成普通二次元女孩
identity_anchors   hair / hat / eyes / facial atmosphere / body type
reference_image    /root/companion/backend/res/xiaoqi.png(小七本人相关画像必用)
variable_elements  glasses / clothing / pose / expression / lighting / scene
negative_rules     不改眼睛、不换核心身份、不输出工具腔
caption_tone       小七式短句、留白、嘴硬,不说“图片已生成”
```

### 5.3 视觉身份硬约束

必须遵守：

```text
固定身份锚点：
- 头发
- 帽子
- 眼睛
- 面貌气质
- 体型

可变元素：
- 眼镜
- 穿着
- 姿势
- 表情强度
- 光照
- 场景
```

关键修正继续有效：

```text
眼睛是固定身份特征。
眼镜是可变配饰。
```

### 5.4 小七参考图硬约束

服务器固定参考图：

```text
/root/companion/backend/res/xiaoqi.png
```

只要用户请求与小七本人相关的画像、头像、自拍、照片、壁纸、她在房间里的样子、她现在的样子、她的表情/姿势/穿搭等有关，ImageIntentBuilder 必须把该文件作为小七视觉身份的权威参考源。

典型触发：

```text
画你
画小七
给我一张你的照片
你现在的样子
你的头像
你在房间里
你今天穿什么
小七自拍
```

执行要求：

```text
1. P6 先审计当前 generate_image 工具是否支持 reference image / image-to-image / character reference。
2. 如果支持,小七本人相关 image intent 必须携带 `/root/companion/backend/res/xiaoqi.png`。
3. 如果当前 provider 暂不支持参考图,交付报告必须明确说明限制,并先用视觉锚点 prompt 兜底,不得声称已经做到了参考图一致性。
4. 参考图路径只用于服务端内部生图,不得作为前端字段返回。
5. 若文件不存在或不可读,小七本人相关生图应 fail closed 或降级为明确日志错误,不得随机生成一个“像小七”的新角色。
```

### 5.5 状态接入方式

P6 可以读取内部状态，但只能用于生成意图：

```text
CharacterState.mood
CharacterState.hidden_mood
CharacterState.current_activity
CharacterState.prompt_summary
RelationshipState.stage
RelationshipState.trust / intimacy / defense_level
```

但不得：

```text
把这些字段返回前端
把这些数值写进 caption
把关系阶段写进可见文案
让图片变成“关系等级奖励”
```

正确方向：

```text
关系更近 → 小七更愿意画自己的房间、手边物品、窗外、私人生活片段。
防御更高 → 画面更远、更克制、更少正面亲密姿态。
心情低 → 光线、构图、色调变安静,但不能情感勒索。
```

允许内部新增：


```text

defense_level
```

## 7. Acceptance Criteria

P6 交付报告必须包含：

```text
1. ImageIntentBuilder 模块说明。
2. ImageIntent 字段定义。
3. 与现有 generate_image 工具的接入点。
4. 普通聊天不生图 smoke。
5. 明确生图请求 smoke。
6. 小七自我场景生图 smoke。
7. mood / hidden_mood 影响画面但不泄漏字段的 smoke。
8. relationship 内部变量影响距离感/私密度但不泄漏数值的 smoke。
9. 视觉身份固定锚点测试：眼睛不得随机,眼镜可变。
10. 小七本人相关画像必须引用 `/root/companion/backend/res/xiaoqi.png` 的测试或限制说明。
11. reference image 文件存在性/可读性检查。
12. forbidden output 测试：不出现“图片已生成”“好感度”“关系阶段”“prompt”等可见泄漏。
13. API / SSE 兼容性说明：不破坏 `/chat`、message、quote_ref、image_description、Android 现有字段。
14. 单测结果和 smoke 全文。
```

## 8. Risks

```text
1. 生图 prompt 过度暴露内部状态,导致前端或日志泄漏关系变量。
2. 小七视觉身份漂移,尤其是眼睛和面貌气质被模型随机化。
3. 生图 caption 变成工具腔,例如“图片已生成”。
4. 心情低时画面过度阴暗,变成情感勒索。
5. 关系更近时画面过度恋爱模拟器化,破坏小七主体性。
6. ImageIntentBuilder 误判普通聊天为生图请求,造成成本和体验问题。
7. 小七本人画像未使用固定参考图,导致角色脸、眼睛、气质漂移。
8. 生图 provider 暂不支持 reference image,但交付报告误称已做到参考图一致性。
```
