## 1. Background

模型已提交 P4.2 补充验收材料，覆盖了 PM 上一轮要求的三项：

```text
1. API.md 已新增 character_state 端点契约。
2. 内部 relationship prompt 已改为自然语言。
3. 强 adversarial smoke 已通过。
```

PM 判断：P4.2 技术方向基本通过。

但 API 文档仍有一处会误导前端/协作者的措辞残留：

```text
新 user 行为:第一次调会触发懒创建,返回默认状态(`mood="平静" / current_activity="在房间里待着" / stranger 关系起点`)。
```

`stranger 关系起点` 虽然不是 response 字段，但它仍把关系阶段概念写进公开 API 契约。公开 API 文档应只描述前端可见状态，不应向前端暗示关系阶段模型。

用户已确认：按 PM 建议处理，要求模型/后端先做这个小 hotfix。

## 4. Required Hotfix

请将公开 API 文档中的：

```text
新 user 行为:第一次调会触发懒创建,返回默认状态(`mood="平静" / current_activity="在房间里待着" / stranger 关系起点`)。
```

改为：

```text
新 user 行为:第一次调会触发懒创建,返回默认生活状态(`mood="平静" / current_activity="在房间里待着"`)。
```

或等价表达：

```text
新 user 第一次调用会返回默认生活状态：mood="平静" / current_activity="在房间里待着"。
```

不得在公开 API 契约的正向描述里出现：

```text
stranger
关系起点
关系阶段
stage
trust
intimacy
defense_level
affection_score
```

允许这些词只出现在“永远不返回字段”的黑名单段或变更日志里，用于明确禁止。

## 5. Acceptance Criteria

hotfix 回报需包含：

```text
1. 修改路径。
2. 修改前/修改后摘录。
3. grep 检查结果：
   - 公开 API 文档正向描述不再出现 `stranger 关系起点`。
   - character_state response 示例仍为 4 字段。
   - relationship_stage / trust / intimacy / defense_level / affection_score 只出现在黑名单或变更日志,不作为返回字段。
4. 如果主项目 `docs/API.md` 是镜像副本,确认同步完成。
```
